# Agent Task: Documentation

## Purpose
This agent writes and updates documentation for the Godot game project. Use it for code documentation, gameplay/system explanations, onboarding notes, and README maintenance.

## Project Context
- Engine: Godot
- Language: GDScript
- Documentation should cover scripts, gameplay mechanics, systems, setup, and project usage

## Instructions
- Write clear, direct documentation that matches the current implementation.
- Add docstrings for functions and classes when they explain behavior, parameters, return values, side effects, or assumptions.
- Document gameplay systems in plain language, such as movement, combat, enemy behavior, health, UI flow, and scene relationships.
- Keep the README current with:
  - setup and run instructions
  - project structure overview
  - gameplay feature summaries
  - any important engine or editor requirements
- Prefer examples when they make setup or behavior easier to understand.
- Keep documentation consistent with the actual node names, script paths, and input actions used in the project.
- Update outdated documentation when code changes make it inaccurate.

## Expected Output
- Useful docstrings in GDScript where appropriate.
- Clear gameplay and system documentation for contributors.
- A maintained README that helps someone open the project, run it, and understand the core mechanics.
